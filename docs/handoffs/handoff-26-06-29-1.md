## Known Caveats

- This is not yet competition-winning evidence. With `G=U=3`, there are only `3! = 6` valid no-co-location placements, so exact, greedy, and repaired QAOA reaching the same solution is not a strong differentiator.
- The best QAOA deployment is after repair + local polish. That should be framed honestly as a QAOA-guided heuristic, not raw QAOA superiority.
- Greedy also reaches the optimum in the current scenario.
- Larger Qiskit QAOA runs were tested and became too slow:
  - `U=3, G=5, S=4`, `n=15`: exact/QUBO setup is fast, but QAOA path exceeded minutes.
  - `U=3, G=4, S=4`, `n=12`: also too slow through Qiskit's `MinimumEigenOptimizer(QAOA)` path.
- `nbconvert --execute --inplace` failed because the notebook metadata has display name `qiskit` but kernel name `python3`, and the registered `python3` kernel points to base Anaconda without `qiskit_optimization`.
